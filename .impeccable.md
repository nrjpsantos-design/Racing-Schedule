## Design Context

### Users
Fãs de automobilismo de todo o mundo que querem acompanhar calendários de corrida de múltiplos campeonatos (F1, MotoGP, WEC, IndyCar, Fórmula E, NASCAR, etc.) em um só lugar. Usam o app para não perder sessões, saber onde assistir e conferir horários no seu fuso horário.

### Brand Personality
Técnica, precisa, confiável. Como uma tela de cronometragem oficial de corrida — sóbria, informativa, sem frescuras. O tom é de dados de telemetria: cada elemento existe por uma razão funcional.

### Aesthetic Direction
- **Tema**: Dark mode apenas. O escuro remete ao ambiente de boxes e night races, e funciona bem para consultas rápidas em qualquer ambiente.
- **Visual**: Limpo, tipográfico, baseado em dados. Cores dos campeonatos como acentos funcionais. Evitar decoração desnecessária — cada pixel deve comunicar informação.
- **Referências**: Telas de timing de transmissão de F1/MotoGP, painéis de dados de corrida, cronometragem esportiva.
- **Anti-referências**: Interfaces genéricas de SaaS, dark mode com gradientes roxos/ciano, glassmorphism, hero metrics decorativos.

### Design Principles
1. **Informação em primeiro lugar**: Layout e hierarquia existem para comunicar dados, não para decorar.
2. **Precisão técnica**: Horários, fusos e status de sessão devem ser absolutamente confiáveis e claros.
3. **Mínima fricção**: O fã quer saber "quando é a próxima corrida" em segundos, não em minutos.
4. **Consistência entre campeonatos**: 26+ campeonatos com estruturas diferentes, mas uma interface unificada.
5. **Escuro com intenção**: O tema escuro não é um default preguiçoso — ele serve ao contexto de uso (telas de timing, boxes, night races).
